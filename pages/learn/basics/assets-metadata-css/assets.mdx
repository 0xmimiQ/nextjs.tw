## Assets

Next.js 可以在<strong>頂層<a href="docs/basic-features/static-file-serving" className="link-code" target="_blank"> `public` 目錄</a> </strong>下提供靜態資源，例如圖片。`public` 目錄中的檔案可以從應用程式的根目錄引用，就像 <a href="/docs/basic-features/pages" className="link-code" target="_blank">`pages`</a> 目錄一樣。

`public` 目錄對於 `robots.txt`、Google Site Verification 和其他任何靜態資源也很有用。請查看 <a href="/docs/basic-features/built-in-css-support" className="link-code" target="_blank" >Static File Serving</a>文件以瞭解更多。

### Download Your Profile Picture

首先，讓我們取得您的個人頭像。

- 下載您的個人頭像，格式為 .jpg（或使用此檔案）。
- 在 public 目錄下創建一個名為 images 的目錄。
- 將圖片保存為 public/images 目錄中的 profile.jpg。
- 圖片尺寸可以大約為 400px x 400px。
- 您可以直接刪除 public 目錄下未使用的 SVG 標誌檔案。

### 尚未優化的圖片

如果是使用 HTML，你可能會像這樣加入你的圖片：

```jsx
<img src="/images/profile.jpg" alt="Your Name" />
```

然而，這也代表了你必須手動處理以下問題：

- 確保圖片在不同螢幕大小下響應式
- 使用第三方工具或庫優化你的圖片
- 只在圖片進入視口時加載

在 Next.js 裡，作為代替提供了 `Image` 元件，去幫你處理上述這些問題。

### Image 元件和 Image 優化

<a href="/docs/api-reference/next/image" className="link-code" target="_blank">`next/image`</a> is an extension of the HTML `<img>` element, evolved for the modern web.

Next.js 預設也支援圖片優化。這使得圖片能夠在瀏覽器支援的現代格式（例如 WebP）下進行調整大小、優化和提供。這樣可以避免將大型圖片發送到視口較小的設備上。此外，它還允許 Next.js 自動採用未來的圖片格式並將它們提供給支援這些格式的瀏覽器。

自動圖像優化可與任何圖像源一起使用。即使圖像由外部數據源（例如 CMS）托管，它仍然可以進行優化。

### 使用 Image 元件

Next.js 在使用者發出請求時，會動態優化圖片，而非在建置時進行優化。這與靜態網站產生器以及僅支援靜態內容的解決方案不同，不論是上傳 10 張還是 10 百萬張圖片，都不會增加建置時間。

圖片默認採用「懶加載」方式，這意味著圖片不在可視區域時不會影響頁面速度。當圖片被滾動到可視區域時才會加載。

圖片總是以避免累積版面位移的方式呈現，這是 Google 將會在搜尋排名中使用的核心網頁要素。

這裡有一個使用 next/image 來顯示我們個人頭像的範例。height 和 width 屬性應該是期望的渲染大小，並具有與源圖像相同的長寬比。

**注意：** 稍後在"拋光佈局（Polishing Layout）"中我們會使用這個組件，現在不需要複製它。

```js
import Image from 'next/image';

const YourComponent = () => (
  <Image
    src="/images/profile.jpg" // Route of the image file
    height={144} // Desired size with correct aspect ratio
    width={144} // Desired size with correct aspect ratio
    alt="Your Name"
  />
);
```
